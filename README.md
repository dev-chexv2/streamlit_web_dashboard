# 🌞 여름 생존 대시보드 (Summer Survival Dashboard)

폭염 속 하루를 데이터로 버티는 Streamlit 앱.
**전기료 폭탄 경보 + 열대야 판정·예약냉방 추천 + 외출 타이밍 판정**을 한 화면에서 제공하고,
AI가 이 모든 결과를 종합한 "오늘의 여름 브리핑"을 생성합니다.

## 빠른 시작

```bash
pip install -r requirements.txt
streamlit run app.py
```

**API 키가 없어도 바로 실행됩니다.** 개발자가 등록해둔 기본 API 키로 바로 동작하며,
그마저 실패하면 내장 샘플 예보(`data/fallback.json`)로 전체 기능이 동작합니다.

## API 키 설정

일반 사용자는 키를 발급받아 입력할 필요가 전혀 없습니다. 개발자가 등록해둔 기본 API 키로
서버 쪽에서 바로 동작하고, 화면(사이드바)에는 키 입력창 자체가 없습니다 — 어떤 API를 쓰는지는
앱의 "ℹ️ 대시보드 안내" 탭에서 확인할 수 있습니다.

## 프로젝트 구조

```
summer_survival/
├── app.py                 # 메인 대시보드 (화면 조립, ℹ️ 안내 탭 포함)
├── utils/
│   ├── weather_api.py     # 기상청 단기예보 호출·파싱 + fallback [9장 API]
│   ├── air_env.py         # 자외선지수 + 에어코리아 대기질      [9장 API]
│   ├── bill_calc.py       # 누진제 요금 + 체감온도 공학 계산   [8장 분석]
│   ├── vision.py          # 고지서·효율라벨 AI 인식 (Gemini)   [11장 이미지]
│   └── briefing.py        # AI 여름 브리핑 생성 (Gemini)        [10장 텍스트, 6장 프롬프트]
├── data/fallback.json     # 내장 샘플 예보 (데모 비상용)
└── .streamlit/secrets.toml.example
```

## 기능 상세

### 💸 전기료 폭탄 경보
- 지난달 사용량 + 에어컨 소비전력·가동시간 → 이번 달 누적 사용량 예측
- 한전 주택용(저압) **누진제** 적용: 하계(7·8월) 완화 구간 300/450kWh 반영
- 그래프에서 누진 구간을 색 밴드로 표시, **구간 돌파 예상일**에 경고 마커
- 고지서 사진을 올리면 AI가 사용량을 자동 인식 (틀리면 직접 수정 가능)
- 에어컨 **에너지효율등급 라벨**(초록 딱지)을 찍으면 소비전력·등급 자동 인식, 4~5등급이면 저효율 경고

### 🌙 열대야 & 예약냉방
- 밤(18시~익일 9시) 최저기온으로 열대야 판정
- 밖 기온이 25°C 아래로 떨어지는 시각까지 **예약냉방 추천 시간** 역산
- 현재 누진 단계 기준 **한계비용(원/kWh)** 으로 오늘 밤 냉방 비용 추정

### 🚶 외출 타이밍 (+ 자외선·대기질)
- 기상청 여름 **체감온도 공식** (Stull 습구온도 근사식) 직접 구현
- 24시간 체감온도를 위험 등급 색으로 시각화, 외출 추천/회피 시간대 판정
- 자외선지수(생활기상지수 API) + 미세먼지·오존(에어코리아 API)까지 종합한 **오늘의 외출 판정** (최악값 지배 방식)
- 공공데이터포털 활용신청 3종: ① 기상청_단기예보 ((구)_동네예보) 조회서비스 ② 기상청_생활기상지수 조회서비스(3.0) ③ 한국환경공단_에어코리아_대기오염정보 — **키는 계정당 하나로 공용**, 미신청 서비스는 샘플값으로 동작

### 📣 AI 여름 브리핑
- 세 모듈의 결과를 종합해 4문장 브리핑 생성 (페르소나 프롬프팅으로 고정된 톤 적용)
- API 키가 없거나 호출 실패 시 규칙 기반 브리핑으로 자동 대체 → **브리핑 칸은 절대 비지 않음**

### ℹ️ 대시보드 안내 탭
- API·데이터 출처, 기본 제공 키 안내, 대시보드 보는 법, 사용 기술스택을 한 화면에 정리

## 시험 대비 보너스

기말고사 범위(7~11장)가 이 프로젝트 코드에 전부 들어 있다. 각 파일 상단 주석에 담당 챕터가 적혀 있으니, 코드 리뷰가 곧 시험공부다.

## 발표(영상) 팁

1. **오프닝**: 실물 고지서를 촬영해 업로드 → 사용량이 자동으로 박히는 장면 (11장 데모)
2. 별도 키 입력 없이 바로 실시간 데이터로 동작하는 걸 보여주기 (9장 데모)
3. 전기료 탭에서 누진 돌파 예상일 짚기 (8장)
4. "ℹ️ 대시보드 안내" 탭에서 API 출처·기술스택 짚어주기 (6장·9장 종합)
5. 와이파이가 죽어도 fallback으로 계속 시연 가능 — 영상 재촬영 걱정 없음

## AI 협업 기록 (12장)

이 프로젝트는 AI(Claude)와 협업해 개발했으며, 보고서에 다음 프로세스를 기술한다:
1. **역할 분리** — 아이디어 선정·요구사항 정의·검증은 팀이, 초기 코드 골격 생성은 AI가 담당
2. **검증 주체는 사람** — AI가 만든 계산 로직(누진제, 체감온도)은 팀이 실제 고지서·기상청 공식과 대조해 검증
3. **이해 없는 코드는 채택하지 않음** — 각자 담당 모듈을 코드 리뷰로 설명 가능해야 통합
4. **반복 개선** — 실행 에러와 실제 데이터를 AI에게 피드백하며 코드를 다듬는 사이클

## 책임 있는 AI 활용 (13장)

- **AI 생성물 표시**: 브리핑에 "AI 생성 / 규칙 기반" 출처를 항상 명시
- **인간 검증 루프**: 고지서·라벨 인식 결과를 사용자가 직접 수정할 수 있는 입력창 제공 (AI 오인식 대비)
- **한계 고지**: 예측은 단순화 가정 기반 추정치임을 화면 하단에 명시
- **데이터 출처 표기**: 기상청·한전·에어코리아 출처를 화면에 표시 (공공데이터 이용허락 준수)
- **키 보안**: API 키는 secrets로 관리, 저장소에 커밋 금지. 화면(사이드바)에는 키 입력창 자체가 없어 실제 키 값이 브라우저로 전송되지 않으며, 서버 쪽 secrets 값만으로 API를 호출

## 알려진 한계 (보고서용)

- 사용량 예측은 "일평균 일정" 가정의 단순 외삽 (요일·기온별 변동 미반영)
- 요금 단가는 코드 상수 (한전 고시 변경 시 `bill_calc.py` 상수 수정 필요)
- 고지서 인식은 촬영 품질에 따라 오차 가능 → 수동 수정 UI로 보완
